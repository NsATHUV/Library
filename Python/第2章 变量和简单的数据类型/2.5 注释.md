在大多数编程语言中，**注释**（comment）是一项非常有用的功能。尽管前面编写的程序只包含Python代码，随着程序规模的增大和复杂性的提高，添加注释是一个良好的实践。注释允许你使用自然语言在程序中添加解释和说明。通过注释，你可以更清晰地阐述解决问题的方法，增强代码的可读性和可维护性。注释不会被编译或执行，它们仅仅是为了程序员的理解而存在。

---
# 2.5.1 如何编写注释

## 单行注释

在Python中，注释使用井号`#`标识。井号后面的内容会被Python解释器忽略，如下所示：

```python
# 向大家问好
print("Hello Python people!")
```

Python解释器会忽略第一行，只执行第二行，输出结果为：

```
Hello Python people!
```

注释是一种用于向代码添加解释和说明的工具，可以提高代码的可读性和可维护性。

## 多行注释

在Python中，多行注释通常使用三个单引号(`'''`)或三个双引号(`"""`)来包围注释内容，如下所示：

```python
'''
这是一个多行注释的示例。
在这里，可以写更多的说明和解释。
注释内容被包含在三个单引号之间。
'''

"""
这也是一个多行注释的示例。
使用三个双引号同样可以包围注释内容。
"""

print("Hello Python people!")
```

多行注释通常用于较长的注释块或文档字符串（docstring），以提供对模块、函数或类的详细说明。在实际编写中，选择单引号或双引号来包围多行注释取决于个人或团队的风格偏好。

# 2.5.2 该编写什么样的注释

编写注释的主要目的是阐述代码要做什么以及是如何实现的。在项目开发过程中，虽然在编写代码时你对各个部分的协同工作了如指掌，但随着时间的推移，你可能会遗忘一些细节。尽管你始终可以通过研究代码来了解各个部分的工作原理，但通过编写注释以清晰的自然语言对解决方案进行概述，可以节省大量时间。

要成为专业程序员或与其他程序员协作，编写有意义的注释是必不可少的。当前，大多数软件是由多名程序员合作编写的，这些程序员可能来自同一家公司，也可能是致力于同一个开源项目的人。训练有素的程序员通常希望代码中包含注释，因此在编写代码时最好养成添加描述性注释的习惯。

新手程序员最值得培养的习惯之一是在代码中编写清晰、简洁的注释。如果不确定是否要编写注释，可以问问自己：在找到合理的解决方案之前，是否考虑了多个解决方案？如果答案是肯定的，就编写注释，对你的解决方案进行说明。相比以后再回头添加注释，删除多余的注释要容易得多。从现在开始，本书的示例将使用注释来阐述代码的工作原理。
